# Neuroscience Context

This document provides conceptual background from neuroscience and cognitive science.

It is **not** making a claim that software continuity systems are brains. The goal is only to explain why certain distinctions in the protocol are natural and useful.

## 1. Working memory

Working memory is limited, active, and fragile.

Protocol analogue:
- live prompt state
- current task focus
- active reasoning scratchpad

Why it matters:
- context windows are effectively working-memory constraints
- continuity exists because working state is not durable by default

## 2. Consolidation

Human memory is not a full replay log. Important information is consolidated into more durable forms.

Protocol analogue:
- promotion from transient state into typed continuity
- compaction into bounded, durable representations

## 3. Episodic vs semantic memory

Episodic memory preserves something like "what happened"; semantic memory preserves more stable knowledge.

Protocol analogue:
- event memory / episodic traces
- typed facts, constraints, decisions, and scars as more reusable semantic state

## 4. Salience and attention

Not every experience is retained equally. Salience, surprise, conflict, and relevance change what survives.

Protocol analogue:
- salience weighting
- prioritization of incidents, constraints, and operational scars
- bounded carryover focused on high-value state

## 5. Replay and reconsolidation

Biological systems replay and reconsolidate memory in ways that reshape future recall.

Protocol analogue:
- compiled continuity
- re-ranking after new supporting evidence
- repair or supersession when better state arrives

## 6. Prediction error and conflict

Brains pay special attention to error, mismatch, and surprise.

Protocol analogue:
- incidents
- contradictions
- conflicts over work claims
- scars derived from failures

## 7. Limits of the analogy

The analogy stops here:

- brains are biological, software systems are engineered
- the protocol should not inherit neuroscientific terminology blindly
- neuroscience is context for design intuition, not proof of correctness
